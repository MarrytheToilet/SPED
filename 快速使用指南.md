## 🚀 启动系统

```bash
python menu.py
```

## 📖 完整工作流程

### 步骤1: PDF处理 (选项 1-3)

```
1. 📤 上传PDF到MinerU
   → 将PDF放入 data/raw/pdfs/
   → 菜单选择 1
   → 记录 batch_id

2. 🔍 查询批次状态
   → 菜单选择 2
   → 输入 batch_id
   → 等待状态变为"done"（5-30分钟）

3. 📥 下载解析结果
   → 菜单选择 3
   → 选择已完成的批次
   → 结果保存到 data/processed/parsed/
```

### 步骤2: 数据提取 (选项 5-7)

```
5. 🧪 测试系统配置（首次使用）
   → 验证API配置
   → 验证LLM连接

6. 📝 提取单个论文
   → 交互式选择论文
   → 选择提取模式（推荐：full）
   → 结果保存到 data/processed/extracted/

7. 🚀 批量提取所有论文
   → 一次性提取所有已解析论文
```

### 步骤3: 数据库管理 (选项 8-11)

```
8. 📥 导入JSON到数据库
   → 批量导入所有JSON文件
   → 保存到 data/artificial_joint.db

9. 📊 查看数据库统计
   → 查看记录数、论文数等

10. 📤 导出Excel
   → 多sheet格式
   → 保存到 data/exports/

11. 📤 导出CSV
   → CSV格式
   → 保存到 data/exports/
```

## 🐛 常见问题

### Q: 上传后如何知道何时完成？

**A**: 定期查询状态

```bash
python menu.py
选择 2 → 输入 batch_id

# 或使用命令行
python scripts/pdf.py status batch_id

# 状态为 "done" 时可以下载
```

### Q: 忘记了batch_id怎么办？

**A**: 查看所有批次

```bash
python menu.py
选择 2 → 直接回车（列出所有批次）

# 或使用命令行
python scripts/pdf.py status
```

### Q: 下载后找不到文件？

**A**: 检查解析结果目录

```bash
ls data/processed/parsed/

# 每个论文一个目录，包含：
# - full.md (完整Markdown，用于提取)
# - images/ (图片)
# - auto/ (其他文件)
```

### Q: 提取时找不到论文？

**A**: 确保已下载解析结果

```bash
python menu.py
选择 12 (查看系统状态)

# 查看 "已解析论文" 数量
# 如果为0，需要先完成PDF处理流程
```

## ⚡ 并发优化：多API Key配置

如果你有多个API账号，可以配置多个API Key分散并发压力：

### 配置方法

在 `.env` 文件中添加备用Key：

```bash
# 主Key（必填）
SILICONFLOW_API_KEY=your_primary_key

# 备用Key（可选，逗号分隔）
SILICONFLOW_BACKUP_KEYS=your_key_2,your_key_3,your_key_4

# 或使用编号后缀格式
SILICONFLOW_API_KEY_2=your_key_2
SILICONFLOW_API_KEY_3=your_key_3
```

### 功能特点

- ✅ **自动轮询**：并发时均匀分配请求到所有Key
- ✅ **故障检测**：连续失败3次自动跳过该Key
- ✅ **自动恢复**：2分钟后自动重试不健康的Key
- ✅ **回退保护**：所有备用Key失效时回退到主Key

### 查看配置状态

```bash
python menu.py
选择 5 (测试系统配置)
# 会显示API Key负载均衡状态
```
